# LidMute 琥珀控制台界面设计

## 目标

保持 LidMute 全部功能与行为不变，将当前纵向堆叠、深色块状界面重构为 A 方案“琥珀控制台”：温暖琥珀与海玻璃色光场、清晰的保护状态、紧凑双列功能区、可扫读的活动时间线，以及完整适配 Liquid Glass 的按钮体系。

## 窗口与层级

- 默认窗口改为约 `980 x 760`，最小尺寸 `900 x 700`，避免当前窄而过高的面板比例。
- 顶部是轻量品牌栏：图标、`LidMute`、一句副标题、Chrome 桥接状态，不使用独立深色卡片。
- 第一视觉层是横跨窗口的状态 Hero：大号中文状态、简短解释、当前保护来源和守卫主开关。
- 第二层为双列卡片：左侧“自动保护”包含夜间时段与模拟合盖；右侧“当前声音”包含音频进程和媒体控制。
- 底部是固定高度活动时间线，事件用图标、中文标题、说明和时间分层，不再直接展示英文枚举作为主标题。

## Liquid Glass 按钮

- 所有操作按钮使用自定义 `LiquidGlassButtonStyle`，不再使用 `.borderedProminent` 或不透明矩形底色。
- macOS 26 使用 `.glassEffect(.regular.tint(color).interactive(), in:)`；旧系统使用 `.ultraThinMaterial`、半透明描边和柔和阴影回退。
- 主守卫按钮采用胶囊形玻璃，开启时仅加琥珀色轻染和发光边缘；关闭时为中性玻璃。
- 模拟合盖/开盖组成双段玻璃控制。可执行项高亮，当前不可执行项仍保留玻璃质感但降低饱和度，不出现系统默认灰色按钮。
- 媒体控制使用圆形玻璃图标按钮；播放/暂停稍大，上一首/下一首对称排列。
- 危险或清理动作使用低饱和暖红文字和透明玻璃，不使用实心红色。
- 悬停时提高高光和描边，按压时缩放至 `0.97`；动画时长约 `0.16s`，不加入持续漂浮动画。

## 色彩与材质

- 背景基色是暖砂色、低饱和青绿和雾蓝的柔和光场，移除当前大面积暗黑遮罩。
- 玻璃卡片保持透明，依靠亮边、局部高光和轻阴影建立层级；卡片之间不叠加多层不透明背景。
- 琥珀色只用于守卫、保护事件和主要动作，青色用于 Chrome/音频状态，中性文字使用深灰蓝。
- 深色系统外观下自动切换为深海蓝灰玻璃，但仍保留琥珀高光，不做纯黑仪表盘。

## 信息呈现

- 守卫状态使用人类可读语句，例如“外放安静，一切正常”“正在保护内建扬声器”“等待合盖或夜间息屏”。
- 夜间时段以单行时间范围和开关呈现，编辑状态才展开两个时间输入框，减少常态视觉噪声。
- 当前声音默认只显示最重要的程序名和状态，PID、Bundle ID、路径放在次级详情行。
- 活动时间线按事件类型映射 SF Symbol 与中文标签；Chrome 标签页保留标题和 URL。

## 验收

- 所有现有功能入口仍存在且可通过辅助功能树识别。
- 主窗口在 `980 x 760` 下不需要纵向滚动即可看到 Hero、双列功能卡和活动时间线。
- macOS 26 实际使用交互式 `glassEffect`，按钮没有系统蓝色实心底。
- 冷启动、悬停、选中、禁用和按压状态均可辨识；按钮状态不依赖颜色单独表达。
- Swift 构建、核心行为测试、Chrome 测试与连续两次 `.app` 打包全部通过。
