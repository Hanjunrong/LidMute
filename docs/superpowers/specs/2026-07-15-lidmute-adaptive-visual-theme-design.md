# LidMute 自适应明暗视觉主题设计

## 目标

在不改变当前窗口结构、布局尺寸和 Liquid Glass 产品识别的前提下，提升 LidMute 在浅色和深色模式下的信息可读性、状态识别速度和视觉层级。

核心目标是让用户第一眼识别：

1. 守卫是否开启。
2. 当前是否处于保护状态。
3. 当前是否有活动音频。

## 当前问题

- 背景渐变和彩色光晕在浅色模式下过于抢眼，削弱文字与卡片边界。
- 所有卡片使用近似相同的玻璃、黑色底色和白色描边，主次层级不明显。
- 组件中大量使用固定白色透明度、黑色透明度和系统 secondary 色，没有统一的明暗模式语义。
- Hero 的守卫状态、状态 pill、音频卡和时间线都使用偏弱的颜色，重要信息不够突出。
- 禁用控件通过整体降低透明度表达，导致浅色和深色模式下都容易变成“灰掉的玻璃”。

## 设计原则

### 1. 保留品牌语言

继续使用 Amber、Sea Glass 和 Mist Blue 作为品牌与状态色，但它们只负责表达语义，不再承担大面积背景色职责。

### 2. 主题 token 统一管理

在 `AmberVisualTheme` 中建立按 `ColorScheme` 解析的语义 token：

- `canvas`：窗口背景。
- `atmosphere`：低饱和背景渐变与光晕。
- `surfacePrimary`：Hero 主卡背景。
- `surfaceSecondary`：普通功能卡背景。
- `surfaceTertiary`：时间线与辅助区域背景。
- `border`：卡片和控件描边。
- `primaryText`、`secondaryText`、`tertiaryText`：正文层级。
- `accent`、`success`、`info`、`danger`：语义色。

组件不再直接写固定的 `.white.opacity(...)` 或 `.black.opacity(...)` 作为主要表面颜色。

### 3. 明确卡片层级

- Hero：最高对比度、最强边界，是唯一的主视觉中心。
- 当前声音：次高对比度，突出实时音频状态。
- 自动保护、模拟测试：标准功能卡。
- 活动时间线：中性且稳定，优先保证事件文字可读性。

卡片仍保持当前零间距和圆角布局约束，不调整 `VisualLayoutMetrics` 的尺寸契约。

### 4. 状态优先于装饰

- Hero 状态徽标使用实色或高对比度语义背景。
- 开启守卫按钮使用明显的 accent 主按钮；关闭状态使用中性按钮。
- 状态 pill 使用轻量色块和对应前景色，不再全部使用 secondary 文字。
- 音频卡明确区分“有活动音频”和“无活动音频”。
- 时间线保留事件色，但提高事件标题、详情和时间戳的层级差异。

## 组件范围

### `AmberVisualTheme.swift`

新增主题 token 和明暗模式解析逻辑；降低浅色背景的彩色光晕强度；为卡片背景和描边提供统一入口。

### `ContentView.swift`

迁移 Header、Hero、MetricPill、AutomationCard、SimulationCard、NowPlayingCard、ActivityTimeline 和 EventTimelineRow 的颜色来源；强化 Hero 状态、实时音频状态和时间线正文层级。

### `LiquidGlassControls.swift`

让主按钮、图标按钮和禁用状态使用主题 token；保持现有交互动画、尺寸和 Liquid Glass API 兼容性。

### `VisualLayoutMetrics.swift`

本轮不修改尺寸常量，仅保留现有布局契约作为回归边界。

## 明暗模式目标

### 浅色模式

- 背景趋向暖灰和低饱和蓝灰。
- 卡片使用稳定的浅色 surface，而不是白色透明层叠加彩色背景。
- 主文字保持深色，辅助文字仍清晰可读。
- Amber 用于主状态和主操作，避免大面积铺色。

### 深色模式

- 背景保持深蓝灰，不使用过亮的白色玻璃叠层。
- Hero 与当前声音卡比普通卡明显抬高一个层级。
- 描边和状态色降低刺眼程度，但状态前景保持清晰。
- 禁用控件保留轮廓，避免完全融入背景。

## 验证方案

1. 使用项目打包脚本构建当前应用。
2. 在 900×680 窗口尺寸下检查浅色和深色模式。
3. 检查 Hero、当前声音、时间线三处是否能快速识别主要状态。
4. 检查无活动音频、守卫关闭、夜间保护中、Chrome 未连接等状态。
5. 运行现有视觉原则检查和核心行为测试。
6. 确认最终 `dist/LidMute.app` 使用最新源码构建，且代码签名验证通过。

## 非目标

- 不改变业务逻辑、事件模型、Chrome 通信和媒体控制行为。
- 不改变窗口整体布局、卡片尺寸、时间线行高和零间距原则。
- 不新增用户设置项或主题切换开关。
- 不在本轮引入新的第三方依赖。
