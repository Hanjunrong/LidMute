# LidMute Aurora Glass 视觉升级设计

## 目标

将用户选定的 A「Aurora Glass」方向应用到 LidMute 主窗口和应用图标，使界面从平面半透明卡片升级为具有折射层次、渐变气氛和苹果式克制感的现代 Liquid Glass 界面。

验收时应能直接看到：

1. 卡片与背景之间有明确但轻盈的空间层级。
2. 不同功能卡拥有协调而不重复的渐变色相。
3. 玻璃高光、描边和阴影共同塑造厚度，而不是依赖单一透明填充。
4. 应用图标和界面功能图标采用统一的层叠玻璃语言。

## 已确认视觉方向

采用 A「Aurora Glass」：

- 背景使用暖琥珀、雾蓝和海玻璃色的低饱和环境光。
- 卡片主体保持高可读性的浅色或深色玻璃基底，在边缘加入柔和折射色。
- 高光集中在左上边缘，阴影落向右下，形成一致的光源方向。
- 色彩用于状态和空间，不使用大面积高饱和霓虹。
- 视觉气质接近苹果原生界面的轻盈、连续与克制，而不是游戏化发光面板。

## 视觉系统

### 背景气氛

`AmberAtmosphere` 保留现有三色品牌语义，但改为多层径向光晕叠加低对比线性渐变：

- 左上暖琥珀光对应主要保护状态。
- 右下海玻璃光对应连接和安全状态。
- 中央雾蓝过渡承担空间纵深。
- 深色模式降低光晕亮度并提升蓝灰基底；浅色模式避免纯白背景。

### 卡片结构

每张卡片由四层组成，并继续遵守当前零间距与独立圆角约束：

1. 语义渐变基底：Hero、自动保护、模拟测试、当前声音和活动时间线使用同一体系中的不同渐变组合。
2. 原生玻璃层：macOS 26 使用 SwiftUI `glassEffect`；较低系统版本使用材质与透明渐变回退。
3. 光学边缘：左上高光描边加右下低对比色边，模拟玻璃厚度。
4. 空间阴影：使用低透明度、较大模糊半径的环境阴影抬升卡片，不产生厚重黑边。

Hero 保持最高层级；当前声音保持次高层级；普通功能卡更克制；时间线保持安静，优先保证长文本可读。

### 控件

按钮继续使用原生交互式 Liquid Glass：

- 主操作使用琥珀 tint，非主操作使用海玻璃或雾蓝 tint。
- 按下状态保留轻微缩放，并增加高光收敛感。
- 禁用状态保留轮廓与材质，不通过整体变灰到不可辨识。
- Chrome 状态、指标 pill 和时间线图标统一使用小型玻璃胶囊或玻璃圆形承载。

## 图标设计

### 应用图标

重绘 `Assets/AppIcon-1024.png` 的生成脚本，采用用户所称的“iOS 27 风格”：以层叠玻璃、清晰深度和简洁几何为核心，而不假定或复制未公开的系统资产。图标使用层叠玻璃超椭圆：

- 外层为暖琥珀到海玻璃的柔和渐变玻璃底。
- 内层为透明保护盾轮廓，表达“守卫”。
- 盾牌中心组合合盖轮廓与静音声波，表达“合盖时阻止外放”。
- 使用内高光、折射边缘和柔和投影，不使用写实 MacBook 或 Apple 商标。
- 在 Dock 小尺寸下优先保证盾牌和静音语义可辨识。

### 界面图标

新增统一的图标承载组件，为 Header、自动保护、模拟测试、当前声音和时间线图标提供：

- 连续圆角玻璃底座。
- 基于功能语义的两色渐变。
- 内描边和顶部高光。
- SF Symbols 作为清晰的单色前景符号，避免重新绘制难以维护的矢量字形。

## 组件与文件边界

### `Sources/LidMuteApp/AmberVisualTheme.swift`

负责 Aurora 色板、背景气氛、卡片表面层级、光学边缘、阴影和共享图标承载组件。卡片差异通过语义枚举表达，避免各视图散落自定义渐变。

### `Sources/LidMuteApp/ContentView.swift`

为 Hero、自动保护、模拟测试、当前声音和活动时间线指定语义卡片层级；将现有平面图标底座替换为统一的 Aurora 图标组件。业务状态、文字、布局尺寸和交互保持不变。

### `Sources/LidMuteApp/LiquidGlassControls.swift`

统一按钮的高光、描边、禁用态和按压反馈，同时保留 macOS 26 原生 `glassEffect(...interactive())` 路径及旧系统回退。

### `Scripts/render-app-icon.swift`

生成新版 1024×1024 应用图标；继续由现有打包脚本生成 iconset 和 icns，避免引入第三方设计工具或依赖。

### `Scripts/check-visual-principles.sh`

扩展静态视觉契约，验证页面继续使用共享卡片表面和图标组件，并防止回退到散落的平面背景写法。

## 布局与兼容性约束

- 不修改 `VisualLayoutMetrics.cardSpacing = 0`。
- 不修改 Header、Hero、中部卡组和时间线的尺寸契约。
- 保持卡片内容先撑满 frame、再 padding、最后应用卡片背景的修饰器顺序。
- macOS 26 使用原生 Liquid Glass；最低系统版本仍保持 macOS 15，并提供视觉一致的回退。
- 不新增第三方依赖，不修改 Chrome 通信、音频控制、合盖监控和事件存储逻辑。
- 同时支持浅色与深色模式，并保证主要文字、辅助文字和状态色可读。

## 验证与验收

1. 运行视觉原则脚本，确认零间距、尺寸和共享视觉组件约束未回退。
2. 运行核心行为测试，确认视觉修改未改变业务行为。
3. 使用项目规定的 `zsh Scripts/make-app-bundle.sh` 构建并打包，禁止以单独 `swift build` 作为最终结果。
4. 打开最新 `dist/LidMute.app`，分别检查浅色和深色模式。
5. 在最小 900×680 窗口检查卡片边界、文字截断、滚动区域和控件状态。
6. 确认 Hero、当前声音、普通功能卡和时间线存在可感知的层级差异。
7. 检查应用图标在 1024、128、64 和 32 像素下的语义辨识度。
8. 使用 `codesign --verify --deep --strict --verbose=2 dist/LidMute.app` 验证最终包签名。

## 非目标

- 不重排当前信息架构或新增页面。
- 不修改任何业务流程和数据模型。
- 不增加动画背景、持续流光或高耗能视觉效果。
- 不复制系统私有图标或未公开的 Apple 资产。
