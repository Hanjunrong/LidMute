# LidMute 夜间静音与媒体控制设计

## 目标

扩展 LidMute 的保护策略：开启守卫时不改变当前播放或音量；只有真实合盖、模拟合盖，或配置的息屏夜间时段开始时才静音内建扬声器。同时展示当前系统音频进程，并通过 macOS 系统媒体按键提供上一首、下一首、暂停/开始。

## 行为规则

1. 开启或关闭守卫本身不改变当前音量、静音状态或正在播放的程序。
2. 真实合盖和模拟合盖进入保护态，保存当前内建扬声器状态并静音；开盖恢复合盖前音量但继续保持静音，手动关闭守卫恢复保护前捕获的完整状态。
3. 模拟状态重置后，模拟合盖可点击并高亮，模拟开盖不可点击；点击合盖后，合盖按钮置灰，模拟开盖高亮且可点击。
4. 守卫开启时，显示器进入息屏且北京时间位于配置区间内，进入夜间保护态并保存进入夜间前的完整音频状态。时段结束或关闭守卫时恢复该状态。默认时段为 `00:00-08:00`。
5. 夜间策略只对系统外放生效，不修改耳机、蓝牙、HDMI、USB 或其他外接输出。
6. 当前音频卡片展示 CoreAudio 报告的输出进程名称、PID、Bundle ID、路径和活动状态。媒体按钮发送系统级 `NX_KEYTYPE_PREV/NEXT/PLAY` 按键，不假定目标应用支持精确的进程级控制。
7. 背景优先使用当前 SDK 可用的 `glassEffect` 玻璃层；不可用时使用 `.ultraThinMaterial`、渐变光场和高光描边回退。

## 架构

- `ProtectionCoordinator` 扩展为事件来源状态机，区分合盖保护和夜间保护快照，并在守卫开启时忽略音频状态变化。
- `SystemDisplayMonitor` 监听 `NSWorkspace.screensDidSleepNotification` 和 `screensDidWakeNotification`，将息屏状态送入 ViewModel；日程判断使用 `TimeZone(identifier: "Asia/Shanghai")`。
- `NightSchedule` 是 Core 层的纯值类型，负责跨午夜区间判断、默认值和配置值校验，便于无 UI 测试。
- `SystemMediaController` 使用 `CGEvent` 发送 macOS 系统媒体按键；发送失败时记录错误但不影响静音守卫。
- `AppViewModel` 负责定时刷新音频进程、显示器状态和夜间时间边界，`ContentView` 只消费发布状态和触发操作。

## 恢复与异常

- 合盖、夜间、两者重叠时分别维护来源引用；只要仍有任一保护来源，内建扬声器保持静音。
- 关闭守卫清除所有保护来源并恢复守卫开启前的完整状态。
- 夜间时段结束但仍合盖时，不恢复扬声器，合盖保护继续有效。
- 恢复失败保留错误事件，守卫状态进入不可用提示，不虚构恢复成功。
- 未检测到可控制的内建扬声器时，不修改任何外接设备。

## 验收

- 开启守卫前后播放声音和音量不变。
- 模拟合盖、真实合盖会静音；模拟开盖恢复音量但保持静音。
- 夜间配置默认 `00:00-08:00`，息屏进入时静音，时间结束恢复；关闭守卫立即恢复。
- 活动卡片可看到 Chrome 等当前输出进程，四个媒体按钮发送系统媒体控制事件。
- 重置模拟状态后合盖按钮可点、开盖按钮不可点，状态高亮正确。
- 冷启动和打包后的界面背景使用玻璃层，核心行为和 Chrome 测试全部通过。
